# 🎨 LQS 랜딩 페이지 개발 계획

## 📋 프로젝트 개요

**목적**: LQS(Learning Quality System) 프로젝트를 소개하고, 잠재 사용자와 투자자에게 제품 가치를 효과적으로 전달하는 GitHub Pages 랜딩 페이지 제작

**타겟**: 수험생, 학원 원장, 교사, 개발자 커뮤니티, 잠재 투자자

---

## 🏗️ 페이지 구조 및 섹션

### 1. Hero Section (히어로 섹션)
- **목적**: 첫 인상에서 제품의 핵심 가치 전달
- **콘텐츠**:
  - 캐치프레이즈: "정답률이 아닌, 사고의 흔적을 분석합니다"
  - 서브 헤딩: "메타인지와 질문력을 성장시키는 AI 학습 시스템"
  - CTA 버튼: "데모 보기" / "GitHub 보기"
  - 메인 비주얼: 대시보드 목업 이미지 또는 애니메이션

### 2. Problem Statement (문제 정의)
- **목적**: 사용자가 겪는 문제에 공감대 형성
- **콘텐츠**:
  - "왜 같은 유형을 반복해서 틀릴까요?"
  - "정답률만 보여주는 시스템의 한계"
  - 3가지 핵심 Pain Points:
    1. 사고 과정이 데이터로 수집되지 않음
    2. 질문 습관이 시각화되지 않음
    3. 회고/기록이 수동이라 지속 불가
  - 아이콘 + 짧은 설명으로 시각화

### 3. Solution Overview (솔루션 소개)
- **목적**: LQS가 어떻게 문제를 해결하는지 설명
- **콘텐츠**:
  - Value Proposition 강조
  - 핵심 차별점 3가지:
    1. 사고 과정 자동 수집 및 분석
    2. AI 기반 메타인지 점수화
    3. 질문력 성장 시각화
  - 간단한 프로세스 다이어그램

### 4. Key Features (주요 기능)
- **목적**: 구체적인 기능 소개
- **콘텐츠** (6개 카드 형식):
  1. **사고흐름 캡처**
     - 자연어/선택형 사고 입력
     - 아이콘: 🧠
  
  2. **AI 사고 분석**
     - 사고 단계, 오류 유형 자동 분류
     - 아이콘: 🤖
  
  3. **오답 이유 분석**
     - 근본 원인 자동 추출
     - 아이콘: 🎯
  
  4. **질문력 분석**
     - 질문 깊이, 다양성 점수화
     - 아이콘: ❓
  
  5. **자동 회고 생성**
     - 매일 자동 회고문 생성
     - 아이콘: 📝
  
  6. **성장 대시보드**
     - 메타인지/질문력 변화 추적
     - 아이콘: 📊

### 5. User Personas (타겟 사용자)
- **목적**: 다양한 사용자층에게 맞춤 메시지 전달
- **콘텐츠** (3개 탭 또는 카드):
  1. **수험생 (김민하)**
     - "내 사고 패턴이 어떻게 변하는지 추적하고 싶어요"
     - 핵심 니즈 4가지 나열
  
  2. **재도전생 (박지윤)**
     - "회고는 하고 싶은데 귀찮아요"
     - 핵심 니즈 4가지 나열
  
  3. **학원 원장 (최성훈)**
     - "학생 데이터를 자동으로 수집하고 싶어요"
     - 핵심 니즈 4가지 나열

### 6. How It Works (작동 방식)
- **목적**: 사용자 여정 시각화
- **콘텐츠**:
  - Step-by-step 프로세스 (4단계):
    1. 문제 풀이 + 사고흐름 입력
    2. AI가 실시간 분석
    3. 오답 패턴 자동 분류
    4. 성장 리포트 확인
  - 각 단계별 스크린샷 또는 일러스트

### 7. Technology Stack (기술 스택)
- **목적**: 개발자 커뮤니티에 기술력 어필
- **콘텐츠**:
  - Next.js 로고 + 설명
  - Supabase 로고 + 설명
  - OpenAI 로고 + 설명
  - 아키텍처 다이어그램 (간소화 버전)

### 8. Roadmap (로드맵)
- **목적**: 프로젝트 진행 상황 및 비전 제시
- **콘텐츠**:
  - 타임라인 형식
  - MVP (6-8주): 핵심 기능
  - V1 (8-12주): 학원용 대시보드
  - V2 (12주+): 고급 기능
  - 완료/진행중/예정 상태 표시

### 9. Demo / Screenshots (데모 & 스크린샷)
- **목적**: 실제 제품 미리보기
- **콘텐츠**:
  - 대시보드 스크린샷 3-4개
  - 또는 인터랙티브 데모 링크
  - 캡션으로 각 화면 설명

### 10. Call to Action (행동 유도)
- **목적**: 방문자의 다음 액션 유도
- **콘텐츠**:
  - "프로젝트에 기여하기" (GitHub Star, Contribute)
  - "문의하기" (이메일 또는 폼)
  - "뉴스레터 구독" (선택사항)
  - 소셜 미디어 링크

### 11. Footer (푸터)
- **콘텐츠**:
  - 저작권 정보
  - GitHub 링크
  - 문의 이메일
  - 간단한 소개

## 💻 기술 스택 (랜딩 페이지용)

### 옵션 1: 순수 HTML/CSS/JavaScript
- **장점**: 간단, 빠름, GitHub Pages 호스팅 용이
- **단점**: 컴포넌트 재사용성 낮음
- **추천 라이브러리**:
  - Tailwind CSS (스타일링)
  - AOS (Animate On Scroll)
  - Swiper (슬라이더)

### 옵션 2: React + Vite (추천)
- **장점**: 컴포넌트 기반, 유지보수 용이, 확장성
- **단점**: 빌드 과정 필요
- **추천 라이브러리**:
  - React
  - Vite (빌드 도구)
  - Tailwind CSS
  - Framer Motion (애니메이션)
  - React Icons

### 옵션 3: Next.js
- **장점**: SEO 최적화, 이미지 최적화
- **단점**: GitHub Pages 배포 시 추가 설정 필요
- **추천**: 정적 export 사용

**최종 선택**: **React + Vite + Tailwind CSS** (빠른 개발 + 모던한 스택)
